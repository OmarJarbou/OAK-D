دعنا نكون صريحين: الكود الحالي معقد جداً لدرجة أنه يصعب تتبعه وتصحيحه. 7 مناطق، دمج مجموعات، درجات أمان، تمهيد EMA، قفل الاتجاه، مخرج للطوارئ من LiDAR مع معاوضة الارتفاع، كل ذلك يتداخل ويخلق سلوكيات غير متوقعة مثل التوقف المستمر أو التذبذب.

تحليل المشاكل الجذرية (من الكود الذي أرسلته)
الـ STOP المهيمن: ليس بسبب عائق حقيقي، بل لأن DecisionEngine لديه قواعد صارمة جداً:

يتطلب valid_groups (مجموعات مدمجة من المناطق الواضحة) بعرض كافٍ (REQUIRED_CLEAR_WIDTH_M = 0.64 + 2*0.15 = 0.94 متر). في ممر ضيق أو أمام عائق، تختفي المجموعات الصالحة، فيقرر STOP.

حتى مع وجود ممر واضح جانبياً (مثلاً R1 مفتوحة)، إذا لم تندمج مع CENTER لتشكل عرضاً كافياً، يتوقف.

منطق _steer_instead_of_stop يعتمد على narrow_zone الذي يظهر فقط عند فشل جميع المجموعات – متأخر جداً.

التذبذب (يمين-يسار):

DecisionEngine يحسب safety_score لكل منطقة (معادلة معقدة من p25_depth، close_obstacle_ratio، إلخ). أي ضجيج بسيط في العمق يغير الترتيب.

آليات القفل (latch) تحاول منع التذبذب لكنها تزيد التعقيد.

SIDE_PREFER_MARGIN=0.20 يجعل التبديل بين الجانبين صعباً، لكن التذبذب يحدث عندما تكون الدرجات متقاربة (يختار A ثم B ثم A).

زوايا التوجيه (EEPROM) صحيحة يدوياً ولكن الكاميرا لا تستخدمها بشكل صحيح:

الكود يحول المناطق (LEFT, L2, L1, CENTER, R1, R2, RIGHT) مباشرة إلى أوامر GO:LEFT, GO:L2 ... إلخ.

لكن CorridorAnalyzer يقسم الصورة إلى 7 مناطق متساوية العرض بناءً على زاوية الرؤية (HFOV=71.9°). هذا يعني أن المناطق تتوافق مع زوايا متساوية في فضاء الكاميرا.

أما الأردوينو فيستخدم POT_PIN لتحريك المحرك إلى مواضع محددة مسبقاً (LEFT_ADC, LEFT_ADC + (CENTER-LEFT)/3 ... إلخ). هذه المواضع تتوافق مع زوايا إنسانية (مثلاً LEFT = إدارة كاملة لليسار، L2 = 2/3، L1 = 1/3).

الخلل: المناطق المتساوية في الصورة لا تتوافق خطياً مع زوايا التوجيه الميكانيكية بسبب هندسة العجلات واستجابة السيرفو. أنت تحتاج إلى معايرة تربط zone_index بزاوية توجيه حقيقية، لكن الكود الحالي يفترض تطابقاً تاماً.

النظام الصوتي يكرر الأوامر: tts_worker و CommandPublisher و DecisionEngine لكل منهم منطق hysteresis خاص به، مما يؤدي إلى تأخير وتكرار. إضافة إلى أن speak() تقاطع التشغيل السابق (_tts_player.stop()) ثم تضع الأمر في queue، لكن التكرار يحدث لأن stable_command قد يتغير بسرعة.

المبادئ الجديدة:
كاميرا → قرارات اليمين/اليسار فقط: 3 مناطق (LEFT, CENTER, RIGHT) أو 5 (LEFT, L2, CENTER, R2, RIGHT). 7 مناطق غير ضرورية وتخلق ضجيجاً.

LiDAR → مسافة أمامية + STOP حقيقي فقط: لا توجيه جانبي من LiDAR (LIDAR_STEERING_ENABLED=0 ثابت). دور LiDAR هو منع الاصطدام بالأشياء المنخفضة التي لا تراها الكاميرا (كراسي، عتبات).

اختيار الاتجاه: بناءً على أقرب مسافة (p20_depth أو p5_depth) في كل منطقة، مع انحياز خفيف للمركز. لا حاجة لـ safety_score المركب.

منع التوقف الزائد: لا تتوقف إلا إذا كانت المنطقة الأمامية (CENTER) أقل من STOP_DISTANCE_MM وجميع المناطق الجانبية أيضاً أقل من MIN_SIDE_DEPTH_MM (أي لا مفر).

معايرة الزوايا: يجب إضافة طبقة تحويل من zone_index → target_adc بناءً على قراءات فعلية من الأردوينو (أي LEFT_ADC, CENTER_ADC, RIGHT_ADC يتم قراءتها وإرسالها إلى Python). هذا يضمن تطابق الصورة مع التوجيه الفعلي.

TTS: يشتغل فقط عند تغيير الأمر الفعلي الذي يتم إرساله (publisher.publish يعيد True).

python
# ======================== CONFIGURATION SIMPLIFIEE ========================
MIN_DEPTH_MM = 300
MAX_DEPTH_MM = 5000
STOP_DISTANCE_MM = 700          # LiDAR or camera front < 700mm → STOP
MIN_SIDE_DEPTH_MM = 800         # Side path considered "open" if p20 > 800mm
CENTER_BIAS_MM = 200            # Add 200mm to CENTER p20 when comparing to sides

# Command smoothing
CMD_HYSTERESIS_FRAMES = 3
CMD_COOLDOWN_MS = 500

# Zone mapping (3 zones only)
ZONE_NAMES = ["LEFT", "CENTER", "RIGHT"]
ZONE_TO_CMD = {"LEFT": "GO:LEFT", "CENTER": "GO:CENTER", "RIGHT": "GO:RIGHT"}

# ======================== SIMPLE CORRIDOR ANALYZER ========================
def analyze_simple(depth_frame, cfg):
    h, w = depth_frame.shape
    y1 = int(h * 0.40)
    y2 = int(h * 0.95)
    x1 = int(w * 0.20)   # ignore 20% sides to focus on walkway
    x2 = int(w * 0.80)
    roi = depth_frame[y1:y2, x1:x2]
    valid = (roi >= MIN_DEPTH_MM) & (roi <= MAX_DEPTH_MM)
    if not np.any(valid):
        return None, None, None

    # Split into 3 equal vertical zones
    zone_w = roi.shape[1] // 3
    metrics = {}
    for i, name in enumerate(ZONE_NAMES):
        zx1 = i * zone_w
        zx2 = (i+1)*zone_w if i<2 else roi.shape[1]
        z_valid = valid[:, zx1:zx2]
        z_depth = roi[:, zx1:zx2][z_valid]
        if len(z_depth) < 50:   # insufficient data
            metrics[name] = 9999   # treat as far
        else:
            p20 = np.percentile(z_depth, 20)
            # If zone is empty or mostly invalid -> treat as blocked
            if np.mean(z_valid) < 0.15:
                p20 = 0
            metrics[name] = p20
    return metrics, (x1, y1, x2, y2), valid

# ======================== SIMPLE DECISION ========================
def decide_simple(metrics, lidar_front_mm):
    if lidar_front_mm is not None and lidar_front_mm < STOP_DISTANCE_MM:
        return "STOP", f"LiDAR obstacle at {lidar_front_mm:.0f}mm"

    center = metrics.get("CENTER", 0)
    left = metrics.get("LEFT", 0)
    right = metrics.get("RIGHT", 0)

    # If any zone has no data, treat as far (9999) so we don't stop mistakenly
    if center == 0: center = 9999
    if left == 0: left = 9999
    if right == 0: right = 9999

    # Apply center bias (we prefer to go straight if possible)
    center_biased = center + CENTER_BIAS_MM

    # Emergency stop: front blocked AND no side escape
    if center < STOP_DISTANCE_MM and left < MIN_SIDE_DEPTH_MM and right < MIN_SIDE_DEPTH_MM:
        return "STOP", f"Front blocked ({center}mm) and no side path"

    # Normal navigation: pick direction with farthest p20 (biased center)
    if center_biased >= left and center_biased >= right:
        return "GO:CENTER", f"Center clear ({center}mm)"
    elif left > right:
        return "GO:LEFT", f"Left clearer ({left}mm vs {right}mm)"
    else:
        return "GO:RIGHT", f"Right clearer ({right}mm vs {left}mm)"

# ======================== SMOOTHING (frame-based) ========================
class SimpleSmoother:
    def __init__(self, hysteresis=3, cooldown_ms=500):
        self.history = []
        self.hysteresis = hysteresis
        self.last_sent = None
        self.last_sent_time = 0
        self.cooldown = cooldown_ms / 1000.0

    def update(self, raw_cmd):
        now = time.time()
        self.history.append(raw_cmd)
        if len(self.history) > self.hysteresis:
            self.history.pop(0)
        # Most frequent command in history
        stable = max(set(self.history), key=self.history.count)
        # Cooldown: don't send same command too often, but allow changes
        if stable == self.last_sent and (now - self.last_sent_time) < 1.0:
            return None, stable   # No send
        if stable != self.last_sent and (now - self.last_sent_time) < self.cooldown:
            return None, self.last_sent   # Hold previous
        self.last_sent = stable
        self.last_sent_time = now
        return stable, stable
