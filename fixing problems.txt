## المشاكل الجذرية وإصلاحاتها

### 🔴 مشكلة 1 — FLIP_LR يكسر حماية القفل الجانبي (الأخطر)

`FLIP_LR=1` في `.env` يعني أن الكاميرا مقلوبة، فـ `GO:LEFT` يصبح في الواقع "يمين". لكن في `CommandPublisher`:

```python
# الكود الحالي — خاطئ عند FLIP_LR=1
@staticmethod
def _is_left_command(command: str) -> bool:
    return command in {"GO:LEFT", "GO:L2", "GO:L1"}  # هذه يمين فعلياً عند الـ flip!
```

عندما يكون `locked_left=True` (جدار على اليسار الفيزيائي)، الكود يحظر `GO:LEFT` لكن `GO:LEFT` بعد الـ flip تعني "يمين فيزيائي" = الاتجاه الآمن! **النتيجة: يحظر الاتجاه الآمن ويسمح بالخطر.**

```python
# الإصلاح في command_publisher.py
def __init__(self, cfg: WalkerConfig, serial: ArduinoSerial):
    # ... existing code ...
    self._flip_lr = cfg.FLIP_LR

def _is_left_command(self, command: str) -> bool:
    # بعد الـ flip، GO:LEFT يعني يمين فيزيائي
    if not self._flip_lr:
        return command in {"GO:LEFT", "GO:L2", "GO:L1"}
    else:
        return command in {"GO:RIGHT", "GO:R2", "GO:R1"}

def _is_right_command(self, command: str) -> bool:
    if not self._flip_lr:
        return command in {"GO:RIGHT", "GO:R2", "GO:R1"}
    else:
        return command in {"GO:LEFT", "GO:L2", "GO:L1"}
```

### 🔴 مشكلة 3 — `stickiness` تمنع التحول للطريق الأآمن

```python
# في decision_engine.py
if current_m.safety_score >= best_m.safety_score * 0.80:
    best_target = current_zone  # يبقى في المنطقة الحالية إذا كانت 80% من الأفضل
```

80% هامش عالٍ جداً في سياق شخص كفيف يتحرك. إذا ظهر عائق في المنطقة الحالية وانخفضت سلامتها إلى 0.5 بينما المنطقة الأخرى 0.7 — النسبة 0.5/0.7 = 71% أقل من 80% فينتقل، لكن إذا كانت 0.58/0.7 = 83% يبقى! **الإصلاح:**

```python
# خفض من 0.80 إلى 0.70 فقط
if current_m.safety_score >= best_m.safety_score * 0.70:
    best_target = current_zone
```

### 🟡 مشكلة 5 — `_oak_front_mm` تُعطي صفر عند منطقة فارغة

```python
@staticmethod
def _oak_front_mm(analysis: AnalysisResult) -> float:
    for k in ("L1", "CENTER", "R1"):
        v = float(m.p20_depth)
        if v > 0.0:
            vals.append(v)
    if not vals:
        return 0.0  # ← صفر يعني "لا يوجد بيانات"
    return float(min(vals))  # ← لكن min يختار الأصغر، وهذا قد يكون منطقة فيها بيانات سيئة
```

عند `oak_front_mm < DISAGREEMENT_TRUST_THRESHOLD_MM (800)` تثق بالـ OAK حتى لو البيانات غير موثوقة. **الإصلاح:**

```python
@staticmethod
def _oak_front_mm(analysis: AnalysisResult) -> float:
    vals = []
    for k in ("L1", "CENTER", "R1"):
        m = analysis.corridors.get(k)
        if m is not None:
            v = float(m.p20_depth or 0.0)
            # فقط إذا كانت المنطقة تحتوي بيانات كافية
            if v > 0.0 and m.valid_ratio >= 0.14:
                vals.append(v)
    if not vals:
        return 9999.0  # لا بيانات = افترض الطريق آمن، لا توقف بناءً على لا شيء
    return float(min(vals))
```

### 🟡 مشكلة 7 — `_is_free_candidate` تراكم streak ببطء مع أنها تُرسّل FREE بسرعة

```python
# الكود الحالي: يحتاج FREE_STABLE_FRAMES=6 frames ثم في _apply_mode_hysteresis:
if self._last_stable != "FREE" and now - self._last_change_time < 0.2:  # COMMAND_CHANGE_COOLDOWN_S
    return self._last_stable  # يرفض FREE حتى بعد 6 إطارات!
```

يعني: تحصل على 6 إطارات `_is_free_candidate=True` لكن الـ hysteresis يرفض الانتقال لـ FREE بسبب الـ cooldown. **الإصلاح:** اجعل cooldown للـ FREE أقل:

```python
# في _apply_mode_hysteresis
if raw == "FREE":
    self._free_streak += 1
    self._go_streak = 0
    self._go_candidate = ""
    # cooldown أقل للـ FREE لأنه أآمن من GO
    free_cooldown = self._command_change_cooldown_s * 0.5
    if (self._last_stable != "FREE" and 
        now - self._last_change_time < free_cooldown):
        return self._last_stable, self._free_streak
    if self._last_stable != "FREE":
        self._last_stable = "FREE"
        self._last_change_time = now
    return self._last_stable, self._free_streak
```
